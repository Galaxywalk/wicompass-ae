\section{Artifact Appendix}

\subsection{Abstract}
This artifact supports the MobiCom 2026 paper ``WiCompass: Oracle-driven Data Scaling for mmWave Human Pose Estimation.''  It includes source code, processed datasets, pre-trained VQ-VAE and pose-estimation weights, cached logs/intermediate results, and notebooks/scripts for reproducing the paper's results.  The review workflow has three levels: (1) regenerate paper figures from released logs, (2) recompute results from released weights and intermediate data, and (3) run training from scratch, with smoke-test commands provided as quick sanity checks for the full training entry points.

\subsection{Description}

\subsubsection{How to access}
The anonymous review repository is \url{https://github.com/MobiCom26AE/WiCompass}. The workspace can be downloaded from the Zenodo archive at \url{https://zenodo.org/records/20907837}, or from the repository README's Google Drive and Baidu Netdisk archive links using the supplied tools. After extraction, the setup tool creates repository-local links for \texttt{logs}, \texttt{model\_zoo}, and \texttt{datasets/*}.

\subsubsection{Hardware dependencies}
A CUDA-capable GPU is strongly recommended. Figure reproduction can run on a smaller machine after cached logs are present; Level 2 and full training benefit from multiple GPUs. The authors validated the artifact with 4 NVIDIA RTX 4090 GPUs.

\subsubsection{Software dependencies}
The artifact uses Linux, Conda, Python 3.11, CUDA 12.x, PyTorch 2.x, FAISS GPU, Jupyter, rclone, and the PointNet++ CUDA extension built from source.

\subsubsection{Data sets}
The workspace includes processed AMASS poses, MMFi, mmBody, WiCompass real-world data, RF-Genesis simulation data, cached encoded tokens, KNN outputs, and sampled token/pose intermediates.

\subsubsection{Models}
The released model artifacts include VQ-VAE pose-tokenizer checkpoints and PointTransformer human-pose-estimation checkpoints for cached, intermediate-recompute, and full-training workflows.

\subsection{Installation}
\begin{lstlisting}[language=bash]
conda create -n wicompass python=3.11 -y
conda activate wicompass
pip install --no-build-isolation chumpy
pip install -r requirements.txt
pip install -e .
git submodule update --init --recursive
cd submodules/Pointnet2_PyTorch/pointnet2_ops_lib
pip install -e . --no-build-isolation
cd ../../..
WS=~/wc_ws
DL=tools/download_workspace.py
SETUP=tools/setup_workspace.py
python $DL --workspace $WS --minimal
python $SETUP --workspace $WS
python -m ipykernel install --user \
  --name wicompass --display-name WiCompass
\end{lstlisting}
Omit \texttt{--minimal} for full retraining or full pipeline inspection.

\subsection{Experiment workflow}
The artifact supports three review levels.  Level 1 is the recommended fast path: use the supplied logs, tokens, KNN outputs, sampled poses, model weights, and notebooks to regenerate the paper figures.  Register the kernel once and execute the notebooks in the VQ-VAE, KNN coverage, simulation scaling, real-world scaling, and pilot-study experiment folders.
\begin{lstlisting}[language=bash]
python -m ipykernel install --user --name wicompass \
  --display-name WiCompass
# Open the notebooks with the WiCompass kernel, or execute them with nbconvert.
# Key notebooks: experiments/vqvae_microbenchmark/vqvae_performance.ipynb,
# experiments/knn_coverage/{intra_knn_distribution,knn_coverage}.ipynb,
# experiments/simulation_scaling/simulation_results.ipynb,
# experiments/real_world_scaling/real_world_paper_figure.ipynb,
# and experiments/pilot_study/*/*paper_figure.ipynb.
\end{lstlisting}
Level 2 recomputes representative intermediate results from released weights and data caches.  Reviewers can re-evaluate VQ-VAE checkpoints, regenerate MMFi/MMBody token files, recompute selected KNN coverage, run PPS/random sampling, and decode sampled tokens to SMPL-X poses.
\begin{lstlisting}[language=bash]
cd experiments/vqvae_microbenchmark
python recover_training_results.py
python batch_evaluate_models.py
cd ../..
python src/wicompass/evaluation/encode_datasets_into_tokens.py \
  --dataset MMFi --dataset-type mmfi
python src/wicompass/evaluation/encode_datasets_into_tokens.py \
  --dataset MMBody --dataset-type mmbody
python src/wicompass/knn_coverage/knn_coverage.py \
  --dataset-A logs/wicompass/encoded_tokens/BMLmovi-BMLrub-CMU-GRAB-KIT-MOYO-MoSh-PosePrior-WEIZMANN_tokens.h5 \
  --dataset-B logs/wicompass/encoded_tokens/MMFi_tokens.h5 \
  --metric cosine --k 2 --multi-gpu --sample-ratio 0.01 \
  --output /tmp/wc-ae/knn/mmfi_k2
python src/wicompass/token_space_sampling/pps_sampling.py \
  --A-path logs/wicompass/encoded_tokens/BMLmovi-BMLrub-CMU-GRAB-KIT-MOYO-MoSh-PosePrior-WEIZMANN_tokens.h5 \
  --budget 500 --k 8 --cap-quantile 0.9 --multi-gpu \
  --out-dir /tmp/wc-ae/pps
python src/wicompass/token_space_sampling/convert_sampled_tokens_to_poses.py \
  --tokens-npy /tmp/wc-ae/pps/capped_pps_selected_vectors.npy \
  --model logs/vqvae/vqvae_tokennum16_tokenclass64/best_model.pth \
  --config src/wicompass/configs/joint_vae_base_tokennum16_tokenclass64.json \
  --output-dir /tmp/wc-ae/poses
\end{lstlisting}
Level 3 is the full reproduction path.  It starts from the dataset components, optionally repeats token sampling and RF-Genesis simulation, and retrains pose-estimation models for simulation scaling, real-world validation, and pilot studies.  This path is GPU-expensive and is intended for reviewers with sufficient compute.
\begin{lstlisting}[language=bash]
python src/wicompass/token_space_sampling/random_sampling.py \
  --A-path logs/wicompass/encoded_tokens/BMLmovi-BMLrub-CMU-GRAB-KIT-MOYO-MoSh-PosePrior-WEIZMANN_tokens.h5 \
  --budget 40000 --seed 123 --out-dir logs/wicompass/sampled_tokens/benchmark
python src/wicompass/token_space_sampling/pps_sampling.py \
  --A-path logs/wicompass/encoded_tokens/BMLmovi-BMLrub-CMU-GRAB-KIT-MOYO-MoSh-PosePrior-WEIZMANN_tokens.h5 \
  --budget 40000 --k 8 --metric cosine --cap-quantile 0.9 \
  --seed 123 --multi-gpu --out-dir logs/wicompass/sampled_tokens/pps
python experiments/simulation_scaling/train_simulation_batch.py --list
python experiments/simulation_scaling/train_simulation_batch.py
python experiments/simulation_scaling/train_simulation_batch_different_size.py
python experiments/real_world_scaling/train_real_world.py
# Pilot-study retraining scripts live under experiments/pilot_study/*/.
\end{lstlisting}

\subsection{Evaluation and expected results}
The expected result for Level 1 is trend-level agreement with the paper's figures.  The expected result for Level 2 is numerical agreement within floating-point tolerance for model evaluation and token/intermediate recomputation, not byte-for-byte equality for every file.  Full Results Reproduced should be judged on the listed key claims: VQ-VAE reconstruction/tokenization behavior, improved latent-space coverage, simulator data-scaling trend, real-world validation trend, and pilot-study trends.

Reviewers should expect exact agreement for cached-result figure generation, close numerical agreement for deterministic model evaluations and intermediate artifacts, and trend-level agreement for full retraining, subject to standard GPU floating-point behavior.  Full retraining is supported by the released scripts and data; it is included for completeness and for the Results Reproduced badge, but it requires substantially more GPU time than the cached and intermediate recomputation paths.

\subsection{Experiment customization}
Reviewers can vary notebook inputs, KNN parameters such as \texttt{k}, metric, and sample ratio, PPS budgets, and training scripts under \texttt{experiments/} when sufficient GPU time is available.

\subsection{Notes}
The artifact has short cached-log and intermediate-recompute paths for AE review, plus full from-scratch training paths for complete reproduction. Full retraining is intentionally much more expensive than the standard review workflow.
